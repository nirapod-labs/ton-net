# ton-net v0.1.0 implementation plan

The first milestone. It builds the bottom of the stack and the first useful slice:
the TL codec, the ADNL-over-TCP transport, the liteserver read layer, the config
loader, and the Node binding. Reads are not proof-verified in this release and are
typed so a caller cannot mistake them for verified state. It ships as a git tag and,
if published, only as an `-alpha` pre-release (NET-ADR-004).

The ADNL transport and the liteserver read path were already exercised against a
mainnet liteserver in [spikes/adnl-feasibility](../../spikes/adnl-feasibility);
this milestone rebuilds that path as a documented, tested library on the crate
lines of NET-ADR-003.

## Delivers

From Node and from Rust: connect to a mainnet liteserver over ADNL, read the
current masterchain head, and read an account's raw state, with the results decoded
from the wire and returned as typed, explicitly-unverified values.

What a consumer can do after v0.1.0:

- Load a mainnet config, or parse one from `global.config.json`.
- Open a client to a liteserver, completing the ADNL handshake.
- Call `masterchain_info` and get a fully decoded block id (workchain, shard,
  seqno, root and file hashes).
- Call `account` for an address and get the raw account state bytes the server
  reported, with the proof bytes retained but not checked.

What a consumer cannot yet do, and the release says so: trust any of it without the
server, parse an account's balance or code (that needs the cell engine, v0.2.0),
resolve an address through the DHT (v0.5.0), or run a get-method (v0.7.0).

## Scope

In scope:

1. **TL codec** over `tl-proto`: constructor tags, boxed and bare types, `bytes`,
   `vector`, `int256`, and the specific schema types this milestone reads.
2. **ADNL over TCP**: the 256-byte handshake, the session ciphers, and the stream
   framing, as a sans-I/O core plus a native tokio transport.
3. **Liteserver read layer**: the query wrapper and the read method set below.
4. **Config loader**: parse the `global.config.json` liteserver list; a bundled
   mainnet snapshot.
5. **Address**: parse the raw and user-friendly address forms.
6. **Node binding** over napi-rs: `Config`, `Client`, the two reads, with TypeScript
   types.

The liteserver method set in v0.1.0 (all server-reported, none verified):

- `liteServer.getMasterchainInfo`, `liteServer.getMasterchainInfoExt`
- `liteServer.getTime`, `liteServer.getVersion`
- `liteServer.getAccountState`

Out of scope for v0.1.0, and each has its own later milestone: the cell and BoC
engine, proof verification, block sync, ADNL over UDP, the DHT, the TVM,
`runSmcMethod`, `sendMessage`, and every binding after Node. `getAccountState`
returns the account state as raw BoC bytes; decoding those bytes into a balance,
code, and data is the cell engine's job in v0.2.0. This milestone decodes the TL
envelope, not the cell contents.

## Workspace layout

A Cargo workspace whose crates map to the durable seams of the system design. The
crates move in lockstep (NET-ADR-004); the split is for boundaries, not for
independent versioning.

```
ton-net/
  Cargo.toml                  workspace
  crates/
    ton-net-tl/               TL codec and the schema types this release reads
    ton-net-adnl/             ADNL: the Transport seam, the sans-I/O handshake and
                              framing, the native TCP transport, the async driver
    ton-net-lite/             liteserver read client and response types
    ton-net/                  facade: Config, Address, Client, ServerReported, Error
  bindings/
    node/                     napi-rs binding
  spikes/
    adnl-feasibility/         the proving spike, kept as reproducible evidence
  docs/
```

Dependency direction is one way and shallow: `tl` depends on nothing in the
workspace; `adnl` depends on `tl`; `lite` depends on `adnl` and `tl`; the `ton-net`
facade depends on `lite`; the Node binding depends on the facade. No cycles, and
the facade is the only crate a consumer names.

## Architecture: a sans-I/O core with an async edge

NET-ADR-001 fixes the core as sans-I/O over a transport seam. This milestone applies
that precisely:

- **The protocol core does no I/O.** The TL codec, the ADNL handshake construction,
  and the frame encrypt and decode are pure functions over bytes. They are
  synchronous and fully testable without a socket. The handshake byte layouts and
  the framing that the spike verified become unit-test vectors here.
- **I/O lives behind a `Transport` trait.** A `Transport` moves ordered bytes and
  nothing more. `TcpTransport` implements it over `tokio::net::TcpStream` on native
  targets. A later release adds a WebSocket transport for the browser; the core does
  not change when it does.
- **One async object drives the loop.** `AdnlConnection<T: Transport>` owns a sans-I/O
  channel and a transport, and is the async surface: it completes the handshake and
  runs a query to its answer. `LiteClient<T>` sits above it and speaks liteserver
  methods. The facade `Client` wraps a `LiteClient<TcpTransport>` for the native
  default so a consumer never handles the transport generic.

The core is generic over the transport rather than boxing it, so there is no dynamic
dispatch and no async-trait allocation on the hot path. The browser transport is a
compile-time choice in a different binding crate, not a runtime swap, so a generic
seam costs nothing and stays the honest shape.

## Public API

### Crate root, `ton-net`

```rust
//! ton-net: a direct client for the TON network.
//!
//! This crate connects to a TON liteserver over ADNL and reads chain state without
//! an HTTP indexer in the path. It speaks the wire protocols directly: the TL
//! codec, the ADNL transport, and the liteserver query layer.
//!
//! # Verification status
//!
//! In this release reads are **not** proof-verified. A liteserver's answer is
//! returned as a [`ServerReported`] value, the server's unproven word. Proof
//! verification and block sync arrive in later releases; until then a
//! [`ServerReported`] must not be treated as verified chain state.
//!
//! # Example
//!
//! ```no_run
//! use ton_net::{Client, Config};
//!
//! # async fn run() -> Result<(), ton_net::Error> {
//! let config = Config::mainnet();
//! let mut client = Client::connect(&config).await?;
//!
//! let info = client.masterchain_info().await?;
//! println!("masterchain seqno: {}", info.value().last.seqno);
//! # Ok(())
//! # }
//! ```
#![forbid(unsafe_code)]
#![warn(missing_docs)]
#![deny(rustdoc::broken_intra_doc_links)]
```

### `ServerReported<T>`

```rust
/// A value a liteserver returned without a checked proof.
///
/// In this release the client does not verify the Merkle proofs a liteserver
/// sends, so every read is the server's unproven word. This wrapper keeps that fact
/// in the type: a caller reaches the inner data through [`ServerReported::value`]
/// or [`ServerReported::into_value`], and cannot mistake it for verified state.
///
/// The raw proof bytes the server sent are retained by [`ServerReported::proof`] so
/// a later release can verify them without another round trip.
#[derive(Debug, Clone)]
#[non_exhaustive]
pub struct ServerReported<T> { /* value and proof, private */ }

impl<T> ServerReported<T> {
    /// Returns a reference to the server-reported value.
    ///
    /// The value is not proof-verified; see the type-level note.
    #[must_use]
    pub fn value(&self) -> &T { /* ... */ }

    /// Consumes the wrapper and returns the server-reported value.
    #[must_use]
    pub fn into_value(self) -> T { /* ... */ }

    /// Returns the raw proof bytes the server sent, still unchecked.
    #[must_use]
    pub fn proof(&self) -> &[u8] { /* ... */ }
}
```

### `Client`

```rust

///
/// A `Client` owns one ADNL channel to one liteserver and serves reads over it.
/// Reads in this release are not proof-verified; each returns a [`ServerReported`]
/// value.
///
/// A `Client` serializes its queries: concurrent calls on the same client run one
/// after another over the single channel. Open more than one client for real
/// concurrency.
#[derive(Debug)]
pub struct Client { /* ... */ }

    ///
    /// Tries the config's liteservers in turn until one completes a handshake.

    /// [`Error::Handshake`] if a reached server rejects the handshake.

### The transport seam, from `ton-net-adnl`

```rust
/// A bidirectional byte stream to one liteserver.
///
/// This is the I/O seam of the client. The ADNL protocol logic is sans-I/O: it
/// produces bytes to send and consumes bytes received, and calls a `Transport` to
/// move them. Native targets implement it over TCP with [`TcpTransport`]; the
/// browser target implements it over a WebSocket in a later release.
///
/// A `Transport` moves ordered bytes and does not frame them; ADNL framing is the
/// caller's concern.
pub trait Transport: Send {
    /// Writes the whole buffer to the stream.
    ///
    /// # Errors
    ///
    /// Returns [`TransportError`] if the stream is closed or the write fails.
    async fn write_all(&mut self, buf: &[u8]) -> Result<(), TransportError>;

    /// Reads exactly enough bytes to fill the buffer.
    ///
    /// # Errors
    ///
    /// Returns [`TransportError`] if the stream closes before the buffer is filled.
    async fn read_exact(&mut self, buf: &mut [u8]) -> Result<(), TransportError>;
}
```

`async fn` in a public trait leaves the returned future's `Send` bound unstated,
which the generic driver needs. The crate states it explicitly, either by desugaring
each method to `-> impl Future<Output = ...> + Send` or with the `trait-variant`
macro that generates the `Send` variant. This is recorded as a real detail because
it shapes the trait definition.

## Error taxonomy

One error type for the facade, a closed and `#[non_exhaustive]` enum so variants can
be added before 1.0 without a breaking change. The lower crates carry their own
narrow errors (`TransportError`, a TL `DecodeError`, an ADNL handshake error) that
the facade maps into this set.

```rust
/// The error type for every fallible operation in this crate.
#[derive(Debug, thiserror::Error)]
#[non_exhaustive]
pub enum Error {
    /// A transport failure: a socket could not connect, read, or write.
    #[error("transport: {0}")]
    Transport(String),

    /// The ADNL handshake with a liteserver failed.
    #[error("handshake failed")]
    Handshake,

    /// A query did not complete within its deadline.
    #[error("timed out")]
    Timeout,

    /// The liteserver returned an error response.
    #[error("liteserver error {code}: {message}")]
    LiteServer {
        /// The liteserver error code.
        code: i32,
        /// The liteserver error message.
        message: String,
    },

    /// A wire value could not be decoded, or a frame checksum did not match.
    #[error("decode: {0}")]
    Decode(String),

    /// An address string was not a valid address.
    #[error("address: {0}")]
    Address(String),

    /// A config could not be parsed.
    #[error("config: {0}")]
    Config(String),
}
```

Two invariants hold across the set, both testable:

- A TL decode of any byte sequence terminates and yields a typed value or
  `Error::Decode`. It never panics and never over-allocates on a hostile length. A
  fuzz target guards this.
- Every call is bounded in time. Absent an answer within the deadline, `Error::Timeout`,
  never a hang.

## Async across the FFI boundary

This milestone exists in part to prove the async design on the cheapest binding
(NET-ADR-004). The shape:

- The Rust `Client` is async on a tokio runtime. napi-rs embeds and manages the
  runtime; a `#[napi]` async method returns a JS `Promise`.
- The values that cross the boundary are small in v0.1.0, which keeps the numeric
  mapping honest:
  - `seqno` and `workchain` fit a JS number and cross as numbers.
  - `shard` is a `u64` and crosses as a lowercase hex string, since a JS number
    cannot hold it without loss.
  - `root_hash`, `file_hash`, and the raw account `state` cross as byte buffers.
  - No coin or balance value crosses in v0.1.0, because the account state is not yet
    decoded. When it is (v0.2.0), amounts cross as decimal strings, not numbers.
- The `Client` holds its connection behind an async mutex, so two overlapping calls
  from JS serialize over the one channel rather than corrupting the stream. A
  concurrent, query-id-multiplexed client is a later enhancement, noted below.

## The Node binding surface

The TypeScript a consumer sees, generated by napi-rs from the Rust binding:

```ts
export class Config {
  static mainnet(): Config
  static fromJson(json: string): Config

export interface MasterchainInfo {
  workchain: number
  shard: string        // u64 as hex
  seqno: number
  rootHash: Buffer

}

export interface AccountState {
  workchain: number
  shard: string
  seqno: number
  state: Buffer        // raw BoC, not decoded in this release
}

export interface ServerReported<T> {
  value: T
  proof: Buffer

```

The `ServerReported` shape carries into TypeScript on purpose: the field a caller
wants is one level in, behind a name that says it is unverified.

## Documentation standard

Every public crate in the workspace is held to this. It is a gate, not a
preference: CI fails the build if any of it regresses.

- **Enforced lints.** Each public crate sets `#![warn(missing_docs)]` (denied in CI)
  and `#![deny(rustdoc::broken_intra_doc_links)]`. `cargo doc` builds clean with no
  warnings.
- **Every public item is documented**: modules with `//!`, and every public type,
  trait, function, method, and field with `///`. The first line is a single summary
  sentence in the third person present ("Returns...", "Connects..."); detail follows
  a blank line.
- **`# Examples`** on every public entry point, and examples are doctests that
  compile and run under `cargo test --doc`. A network example is marked `no_run`.
- **`# Errors`** on every function that returns `Result`, naming which [`Error`]
  variants it returns and when.
- **`# Panics`** on any function that can panic. The core crates target zero: the TL
  decoder returns an error rather than panicking on any input, and that is a tested
  invariant.
- **`# Safety`** on any `unsafe` function. The `tl`, `adnl`, `lite`, and facade
  crates set `#![forbid(unsafe_code)]`. Only the Node binding may use `unsafe` at the
  FFI edge, and each use is documented and justified.
- **Intra-doc links** for every type or function named in prose, so the docs
  navigate and the linkage is checked.
- **API Guidelines** (the Rust `C-*` guidelines): `#[non_exhaustive]` on [`Error`]
  and on the response structs that will gain fields; `#[must_use]` on constructors
  and pure accessors; `Debug` on every public type; no private type in a public
  signature; conventional naming for conversions and accessors.
- **MSRV** is documented in each crate and checked in CI. The workspace targets a
  recent stable Rust, because `async fn` in traits (used by the transport seam) needs
  1.75 or newer; the exact floor is pinned in CI.

## Testing and conformance

Three layers, matching [docs/conformance.md](../conformance.md) for the families this
milestone touches (TL and handshake).

- **Hermetic unit tests**, no network, run on every commit:
  - TL: round-trip encode and decode for each schema type; the constructor ids
    cross-checked against the schema magics; a decode fuzz target that must not panic
    or over-allocate.
  - ADNL: the handshake packet and the frame encrypt and decode against fixed
    vectors, including the byte layouts the spike verified; a checksum-mismatch case
    that must be rejected.
  - Config: parse a sample `global.config.json`; reject malformed input.
  - Address: parse the raw and user-friendly forms; reject a bad CRC16.
- **Network integration tests**, marked `#[ignore]` and run in a dedicated CI job
  with network access, not in the hermetic run: connect to a mainnet liteserver, call
  `masterchain_info` and assert workchain `-1`, shard `0x8000000000000000`, and a
  nonzero seqno; call `account` for a known active account and assert a nonempty
  state buffer.
- **The Node gate test**: from Node, `connect`, `masterchainInfo`, and `account` for
  a known account, asserting the shapes and that the calls are real Promises. This is
  the milestone's definition of done for the async path.

Conformance vectors for TL and the handshake are generated against the reference
node and the schemas, and are checked in the hermetic run so they are anchored to
TON, not to this implementation.

## Dependencies

The v0.1.0 subset of NET-ADR-003, per crate. Versions are pinned in `Cargo.toml`;
the lines below name the crate, not the exact patch.

- **ton-net-tl**: `tl-proto`.
- **ton-net-adnl**: `curve25519-dalek` (the ed25519-to-montgomery conversion and the
  DH), `sha2`, `aes`, `ctr`; and for the native transport, `tokio` (features `net`,
  `io-util`, `rt`, `time`, `sync`, `macros`). The spike proved these interoperate.
- **ton-net-lite**: the workspace crates above, plus `thiserror`.
- **ton-net** (facade): `serde` and `serde_json` for the config, `thiserror` for the
  error type. CRC16 for addresses is hand-written, as CRC32 was in the spike, rather
  than pulling a dependency.
- **bindings/node**: `napi`, `napi-derive`, with napi-rs's tokio integration.

No HTTP client is a dependency. Fetching `global.config.json` is the caller's to do;
the core parses a string. This keeps the core free of a transport it does not need
and matches the sans-I/O stance.

## Sequence

Ordered steps, each ending in something checkable. A step does not start until the
one before it is green.

1. **Workspace and CI skeleton.** The five crates, the workspace `Cargo.toml`, and CI
   with `fmt`, `clippy` (warnings denied), `cargo doc` (warnings denied), and the
   doctest run. Exit: an empty workspace builds clean and the doc gate is live.
2. **ton-net-tl.** The codec and the schema types for this milestone, with round-trip
   tests, the id cross-check, and the decode fuzz target. Exit: every type round-trips
   and the fuzz target runs clean.
3. **ton-net-adnl, sans-I/O.** The handshake construction and the frame encrypt and
   decode as pure functions, against the spike's verified vectors. Exit: the vectors
   pass and a tampered frame is rejected, with no socket in the tests.
4. **ton-net-adnl, transport.** The `Transport` trait, `TcpTransport`, and the
   `AdnlConnection` driver. Exit: an ignored network test completes a handshake with a
   mainnet liteserver.
5. **ton-net-lite.** The `LiteClient`, the read method set, and the response decode.
   Exit: an ignored network test reads a plausible masterchain head.
6. **ton-net facade.** `Config` with the bundled mainnet snapshot, `Address`, the
   `Error` enum, `ServerReported`, and `Client` with `connect`, `masterchain_info`,
   and `account`. Exit: the hermetic suite is green and an ignored network test reads
   an account's raw state.
7. **bindings/node.** The napi crate exposing `Config` and `TonClient`, the generated
   TypeScript types, and the Node gate test. Exit: the gate test passes, proving the
   async path across the FFI boundary.
8. **Release.** Tag `v0.1.0`. If published, publish the Rust crates to crates.io and
   the Node binding to npm as `0.1.0-alpha.1` on the `alpha` channel (NET-ADR-004).

## Decisions made in this plan

- **Generic over the transport, not boxed.** The core takes `T: Transport` so there
  is no dynamic dispatch; the browser transport is a compile-time choice in another
  crate. The facade hides the generic behind a concrete `Client`.
- **The account is returned as raw bytes.** With no cell engine in v0.1.0, decoding
  the account is out of scope; the honest surface is the raw BoC the server sent,
  wrapped in `ServerReported`.
- **Unverified reads are a type, not a method name.** `ServerReported<T>` forces a
  caller to acknowledge the value is unverified, and stays the marker when
  verification lands and a verified path is added beside it.
- **A single-connection, serialized client** in v0.1.0. Correct and simple; the
  concurrent, query-id-multiplexed client is a later enhancement, not a v0.1.0
  requirement.
- **Config parsing is HTTP-free.** The core parses a string; fetching is the caller's.

## Open questions

- **tl-proto and the nested `bytes`-of-object pattern.** `liteServer.query` carries a
  serialized boxed object inside a `bytes` field, and `adnl.message.query` carries the
  liteServer query inside another. This needs confirming as a clean `tl-proto`
  expression early in step 2; the spike did it by hand, so the fallback is a thin
  manual wrapper if the derive path is awkward.
- **`async fn` in the public `Transport` trait and its `Send` bound.** Resolved in
  the trait definition either by desugaring to `impl Future + Send` or with
  `trait-variant`; the choice is settled in step 3.
- **The napi numeric mapping** for values beyond v0.1.0 (coin amounts) is decided now
  as decimal strings, but is only exercised once the cell engine lands.

## Definition of done

v0.1.0 is done when:

- From Node, a test connects to a mainnet liteserver, calls `masterchainInfo` and
  gets workchain `-1` with a nonzero seqno, and calls `account` for a known active
  account and gets a nonempty raw state, all as resolved Promises across the FFI
  boundary.
- The hermetic unit suite, the TL and handshake conformance vectors, `fmt`, `clippy`
  with warnings denied, and the doc gate (`missing_docs` denied, no broken intra-doc
  links, doctests passing) are all green.
- The network integration tests pass in their dedicated CI job.
- `v0.1.0` is tagged. Any registry publish is `0.1.0-alpha.1` on the `alpha` channel,
  marked not for production.
