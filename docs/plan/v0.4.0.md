# v0.4.0: the cell engine to full capability

The read path is complete and trust-minimized as of v0.3.0. What the library
cannot yet do is produce a cell it did not receive: `Slice` is load-only,
`Cell::from_parts` is crate-private, and no builder of any kind exists. That one
gap blocks the write path (v0.5.0), the virtual machine (v0.10.0), and this
milestone's own gate. v0.4.0 closes it and brings the cell, slice, builder,
dictionary, usage, Merkle, and bag-of-cells surfaces to full capability, the
node-scale bag operations included, each held to a fact about mainnet rather
than to another implementation.

The cell engine is a general TON library, not the servant of any one consumer.
Capabilities were once sorted by whether a wallet needs them, which excused the
node and indexer scale bag operations (a streaming serializer for a bag too large
to hold in memory, a random-access view that hash-verifies without materializing,
lazy per-reference loading, and the structure-aware compression forms) into a
later milestone. That sort was wrong: the library's surface is not bounded by one
caller, so those operations are in scope here. The prefix dictionary returns for
the same reason, gated by a captured cross-implementation vector since no mainnet
fixture holds one.

Governing decision: [NET-ADR-008](../adr/NET-ADR-008-scope-and-write-path.md)
names the cell engine as the critical path.

## Gate

Two properties define the milestone as releasable:

- A Merkle proof built by this crate verifies in the existing proof engine.
- Every dictionary variant round-trips against captured mainnet fixtures: a real
  dictionary is read, enumerated, rebuilt from scratch, and required to reproduce
  the identical root hash.

Both hold. The dictionary oracle is the load-bearing one, because TON dictionary
construction is canonical: the label encoding must be the shortest form, and a
wrong choice parses cleanly while hashing differently. Rebuilding a real
dictionary and demanding a byte-exact root hash tests that canonicalization in a
way no round-trip can.

## Reference policy

The feature surface is enumerated from a mature reference client and every line
is written from `block.tlb`, TON's schemas, and mainnet bytes. Taking a feature
list carries no attribution obligation; mirroring code would. No file in this
repository names that project. The coverage claim rests on matching mainnet root
hashes, which is proof, rather than on agreement with a second implementation,
which is not.

## Target module tree

Structure follows visibility, not file length
([NET-ADR-011](../adr/NET-ADR-011-code-structure.md)): a module grows downward
into a directory beside its file when a child earns a specialization, and
`pub(super)` is the currency a child exposes to its parent. A node marked done
exists; a node marked pending is named here so its home is fixed before it is
built.

## Coverage checklist and status

Grouped by deliverable, each with an oracle that is a fact about mainnet.

1. **Builder**: store_uint/int (big and var), coins, bit, slice, builder, ref,
   maybe_ref, address, snake strings, same-bit runs, capacity queries,
   rollback/truncate with atomicity, store_dict. Done.
2. **Slice completion**: preload and peek, skip, to_builder, to_cell, address
   and coins loaders, bits_equal, lexicographic compare, count leading/trailing,
   bounded sub-slices (subslice, load_slice, skip_bits_and_refs). Done.
3. **Plain dictionary**: new, from-root, set (all modes), get, remove, iterate,
   count, min/max, take-min/max, entry-at-or-after, retain, key access by
   int/uint width. Done.
4. **Augmented dictionary**: load, set/get with extra, root extra, combine,
   iterator-extra, fork_extras (traverse-extra), validate. Done.
5. **Prefix dictionary**: `PfxHashmap` with longest-prefix lookup, set, remove,
   integer and bytes keys. Pending. No mainnet structure this client decodes holds
   one, so it is gated by a captured cross-implementation vector: the same entries
   are encoded here and the reference client's bytes are pinned once, and byte
   identity is required.
6. **Usage tree**: node-for-cell, mark, mark-path, prune, note
   (load-notification), with/without-trace switch, combine. Done. A transparent
   usage-cell proxy that records loads without the caller routing them is left to
   the TVM milestone, where the read path it wraps is finalized.
7. **Merkle proof creation**: create_proof, usage proof via UsageTree::prove,
   check through the engine and through virtualization. **skeleton and unwrap
   partial.**
8. **Merkle update**: create, apply, may-apply, validate, combine_updates. Done.
9. **Virtualization**: virtualize, actual and effective level, hash/depth
   clamping, is_virtualized, rebuild_with_refs. Done.
10. **BoC variants**: multi-root, serialization options and flags, CRC-32C
    checksum, reserialize, file_hash, produced hash index, read-only BocView.
    Done. The node-scale bag operations are pending: a random-access view that
    hash-verifies each cell without materializing the tree, a per-reference lazy
    loader with prewarm, a one-pass streaming serializer for a bag too large to
    hold in memory, and the cached-hash serialization form the parser already
    reads. Each is gated on agreeing with the in-memory path on the same tree.
11. **BoC compression**: LZ4, behind the `compress` feature, with a bounds-checked
    and capped decoder on the untrusted boundary (lz4_flex admitted by
    NET-ADR-012). Done. Two further forms are pending: a self-contained
    structure-aware form (dedup graph, topological reorder, forward-reference
    delta encoding, pruned-branch header stripping), and a state-differential form
    that elides a Merkle update's old-state subtree and rebuilds it from a peer
    state cell the caller supplies. Each is gated on a compress-then-decompress
    round-trip against a captured block, the state form against a captured block
    pair.
12. **Cell utilities**: dump (text and bits), hash key, copy (via Clone),
    to_builder, JSON both directions behind the `json` feature (serde_json,
    already admitted by NET-ADR-003). Done.

## The oracle strategy

Every capability is held to captured mainnet bytes where they reach it.

| capability | oracle |
|---|---|
| Builder | build then serialize then parse then identical repr_hash; proptest over generated trees |
| Dictionary write | read a real mainnet dictionary, enumerate, rebuild from scratch, require the identical root hash |
| Augmented dict | a whole block's `account_blocks`, eleven entries over ten forks rebuilt to the validators' own root hash in both orders |
| Merkle proof creation | prune a real block to a path; the proof verifies in the engine and reproduces the server's own proof cell byte for byte |
| Merkle update | rebuild a real block's `state_update` from its two sides and reproduce the cell byte for byte; validate |
| Virtualization | the tree a mainnet proof stands for hashes back to the identity the proof claims |
| Dump | a real block renders with its schema tag legible at the front of the hex form |
| MsgAddress, snake, typed keys | round-trip and spec-layout, pinned against the schemas |

## Progress

- **Step 1**: Builder plus eight Slice additions. A partial write in
  `store_var_uint` found and fixed.
- **Step 2**: the dictionary moved to `ton-net-cell` and gained
  set/remove/iterate; 886 mainnet labels re-encode bit-identically, two
  dictionaries rebuild to their own root hashes.
- **Step 3**: `AugDict` and the `Augmentation` trait over a private `Shape`
  seam shared with the plain dictionary; gated on a block's `account_blocks`
  rebuilding to the validators' root hash in both insertion orders.
- **Step 4**: usage tree and virtualization.
- **Step 5**: Merkle proof creation; a proof this crate builds verifies in the
  engine and reproduces a server proof byte for byte.
- **Step 6**: Merkle update; a real block's `state_update` rebuilds byte for
  byte and validates.
- **Cell and dictionary capability**: dump (text and binary), MsgAddress read
  and write, snake string read and write, dictionary access by integer key,
  slice bit comparison, `Hash` for `Cell` and `to_builder`, and dictionary
  count, min/max, take-min/max, entry-at-or-after, and retain.
- **Capability to parity**: the remaining feature gap closed in one pass, each
  with its own oracle or round-trip test: slice count and bounded sub-slices,
  `Builder::store_dict`, augmented `fork_extras` and `validate` (gated on the
  real `account_blocks` forks), usage `note`/tracing/`combine`, Merkle
  `combine_updates` (gated on a synthetic A-B-C chain across every change
  pattern), virtualization `is_virtualized` and `rebuild_with_refs`, and the BoC
  serialization options, `file_hash`, multi-root, and read-only `BocView`. What
  remains is dependency-gated (JSON, compression), structural, or the release.

## Remaining work

- **First-pass capability: landed.** `store_dict`; slice count leading/trailing
  and bounded sub-slices; augmented `fork_extras` and `validate`; usage `note`,
  the tracing switch and `combine`; Merkle `combine_updates`; virtualization
  `is_virtualized` and `rebuild_with_refs`; BoC serialization options,
  `file_hash`, multi-root, and a read-only `BocView`. Each landed with its own
  oracle or round-trip test and a green gate.
- **Reopened scope**, the node-scale bag operations and the remaining breadth,
  built in dependency order, each slice green on the gate before the next:
  - **R1 enabling primitives.** A `Cell::metadata` accessor for a cell's
    significant hashes and depths and its one-level reference metadata, and a
    trusted-hash constructor that builds an ordinary cell from stored hashes
    without its children present. Both underlie the lazy, view, and large paths.
  - **R2 random-access view and lazy loading.** Turn `BocView` from a header shell
    into a reader that hash-verifies each cell over a windowed read without
    building the tree, and add a per-reference lazy loader with `prewarm`. Gate:
    view and lazy hashes equal the materialized hashes on a large fixture.
  - **R3 streaming serializer.** A one-pass writer for a bag too large to hold in
    memory, importing cells by hash. Gate: its bytes equal the in-memory
    serializer on the same tree.
  - **R4 structure-aware compression.** The self-contained form. Gate: a
    round-trip on a captured block.
  - **R5 state-differential compression.** The form that elides a Merkle update's
    old-state subtree and rebuilds it from a caller-supplied state cell. Gate: a
    round-trip on a captured block pair.
  - **R6 augmented-dictionary breadth.** `AugDict` to plain-`Dict` API parity,
    plus guided traverse-by-summary, structural combine, and the inline iterator.
    Gate: against the real `account_blocks`.
  - **R7 plain-dictionary breadth.** Predecessor lookup, bulk canonical build from
    items, and prefix-subdictionary extraction. Gate: rebuild real dictionaries to
    their root hashes.
  - **R8 builder and slice completion.** `Slice::load_dict`, arbitrary-width
    integers past 128 bits, the variable and external address forms with anycast,
    and slice prefix and suffix predicates. Gate: round-trip against the builder.
  - **R9 serialize completeness.** Write the cached-hash form the parser already
    reads, append into an existing buffer, and a streaming writer. Gate: reparse
    of every form.
  - **R10 proof, update, and metadata edges.** Root-anchored proof check and
    unwrap, a reuse report on update apply, and an ignore-loads control on the
    usage tree. Gate: against captured proofs.
  - **R11 transport utilities.** Base64 and hex bag helpers, and any-cell JSON.
    Gate: a base64 round-trip of a real bag, byte for byte.
  - **R12 prefix dictionary.** `PfxHashmap`, gated by the captured
    cross-implementation vector.
- **Dependency-gated: done.** Cell JSON ships behind the `json` feature over the
  already-admitted serde_json; BoC compression ships behind the `compress`
  feature over lz4_flex, admitted by NET-ADR-012. Both are off by default, so a
  build that wants neither carries neither the code nor the dependency, and the
  single-copy crypto tree NET-ADR-003 fixes is unchanged.
- **Structural: done.** The test tree is consolidated from six binaries to one
  under `tests/cell/`, the dictionary bench is in, and the two large trunks are
  split to match the target tree: `cell.rs` into `cell/{exotic,level,hash}` and
  `boc.rs` into `boc/{header,parse,serialize}`, joining the children (`dump`,
  `json`, `view`, `compress`, `usage/trace`, and the `dict/` tree) already there.
  The shared model stays in each trunk, so the read path, the write path, and the
  view over a bag reach one header and one reader.
- **Feature coverage in CI: done.** The gate and `ci.yml` now run an
  `--all-features` clippy, test, and doc pass over the cell crate, so the `json`
  and `compress` code cannot rot behind its flag.
- **Surface: ongoing.** Each new item is re-exported from the facade as it lands;
  `VERIFY_EPOCH` stays at 1, since v0.4.0 adds the ability to produce and stream
  cells but changes no acceptance rule, and the epoch test confirms the accept set
  is unchanged; the Node binding gains nothing, since the cell engine sits below
  the account-verification surface it exposes; the README describes the crate's
  role, not a feature list, so it needs no change.
- **Release** (operator-gated): the version bump from 0.3.0 to 0.4.0, the
  changelog release-plz generates from the conventional commits, the tag, and the
  publish.

  - **R13 parallel finalization.** Wave-parallel hashing of a large bag above a
    cell-count threshold. This changes no output, only speed, so it lands last and
    is gated on producing hashes identical to the single-threaded finalizer on
    every fixture.

## Out of scope

**A transparent usage-cell proxy that records loads without the caller routing
them** stays with the TVM milestone, where the read path it wraps is finalized.
The explicit `note` covers the need until then.
